# Статистический анализ данных GoFast

## Постановка задачи

Заказчик — **сервис аренды самокатов GoFast**. Цель — проанализировать поведение пользователей с подпиской Ultra и без неё, рассчитать помесячную выручку на каждого клиента и проверить ключевые бизнес-гипотезы с помощью статистических тестов.

## Данные

| Файл | Описание |
|------|----------|
| `users_go.csv` | Пользователи: идентификатор, имя, возраст, город, тип подписки |
| `rides_go.csv` | Поездки: идентификатор пользователя, дистанция (м), длительность (мин), дата |
| `subscriptions_go.csv` | Условия тарифов: цена минуты, цена старта, абонентская плата |

### Тарифы

| Параметр | Free | Ultra |
|---|---|---|
| Абонентская плата | 0 ₽/мес | 199 ₽/мес |
| Старт поездки | 50 ₽ | 0 ₽ |
| Цена минуты | 8 ₽ | 6 ₽ |

## Этапы работы

### 1. Загрузка и предобработка данных
- Считывание трёх CSV-файлов в датафреймы
- Приведение столбца `date` к типу `datetime`, создание столбца месяца
- Проверка и устранение пропусков и дубликатов (найдены дубликаты в `users`)

### 2. Исследовательский анализ данных
- Распределение пользователей по городам (8 городов; лидеры — Пятигорск и Екатеринбург)
- Соотношение подписчиков: 54.4% — Free, 45.6% — Ultra
- Возрастной профиль: большинство пользователей — молодёжь (12–32 года)
- Анализ распределений расстояния и продолжительности поездок

### 3. Объединение данных и визуализация
- Объединение таблиц `users`, `rides`, `subscriptions` через `merge()`
- Сравнительная визуализация дистанций и длительности для Free и Ultra

### 4. Подсчёт выручки
- Агрегация поездок по пользователю и месяцу
- Расчёт помесячной выручки с округлением длительности «потолком» (`math.ceil`)

### 5. Проверка гипотез
| Гипотеза | Тест | Вывод |
|---|---|---|
| Подписчики тратят больше времени на поездки | Двухвыборочный t-тест (правосторонний) | ✅ Подтверждена (p ≪ 0.05) |
| Среднее расстояние подписчиков ≤ 3130 м | Одновыборочный t-тест | ✅ Не опровергнута (p > 0.05) |
| Выручка от подписчиков выше помесячно | Двухвыборочный t-тест (правосторонний) | ✅ Подтверждена (p ≈ 0) |
| Снижение обращений в техподдержку после обновления | — | Требуется **парный t-тест** |

### 6. Анализ распределений (Шаг 7)
- Биномиальная модель: для выполнения плана (100 подписчиков при p=10%) нужно разослать **1 161 промокод**
- Нормальная аппроксимация: вероятность того, что push-уведомление откроют ≤ 399 500 из 1 млн, составляет **~15%**

## Ключевые выводы

1. **Подписчики Ultra — ключевой источник выручки**: они совершают более длинные и регулярные поездки, а их помесячная выручка статистически выше.
2. **Износ самокатов под контролем**: среднее расстояние поездки подписчика не превышает оптимального порога 3 130 м.
3. **Аудитория молодая**: акции и маркетинг стоит ориентировать на возраст 18–32.
4. **Рекомендация**: развивать подписную модель через реферальные программы и акции лояльности.
